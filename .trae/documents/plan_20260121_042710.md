# iOS构建失败修复计划：添加缺失的多语言字段

根据错误日志 `Error: The getter 'totalArrows' isn't defined for the type 'AppLocalizations'`，原因是我在修改 `AnalysisScreen` 时使用了 `l10n.totalArrows`，但在 `AppLocalizations` 抽象类及其实现类中**忘记添加这个字段**。

虽然我在 `AnalysisScreen` 的代码中使用了 `value: stats.totalArrows.toString()`，但 `label` 参数我尝试使用 `l10n.totalArrows`。

## 修复步骤

1.  **修改 `lib/l10n/app_localizations.dart`**：
    *   在 `AppLocalizations` 抽象类中添加 `String get totalArrows;` 定义。

2.  **修改 `lib/l10n/app_localizations_zh.dart`**：
    *   实现 `String get totalArrows => '总箭数';`。

3.  **修改 `lib/l10n/app_localizations_en.dart`**：
    *   实现 `String get totalArrows => 'Total Arrows';`。

## 验证

修复后，`AnalysisScreen` 中的 `l10n.totalArrows` 调用将不再报错，iOS 构建应该能够成功。
